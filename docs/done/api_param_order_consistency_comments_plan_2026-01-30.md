## ゴール

- primitive / effect / preset の引数一覧に対して、以下を md 内に可視コメントとして残す。
  - 命名が揺れている箇所（例: `center` vs `cx/cy`, `keep_original` vs `keep_mode` など）
  - 順序ルールが揺れている箇所（例: `seed` の置き場所、transform 系の並び など）
  - 直すならどう直すか（候補を 1〜2 案に絞る）

## 非ゴール

- 実装（Python コード側）の引数名/順序の変更はしない。
- 互換ラッパー/シムは作らない（必要になったら別タスク）。

## コメントの書き方（案）

- 各 op の下に `> コメント: ...` を追加する（引数リストはそのまま保持）。
- 併せて、先頭に「全体ルール案」セクションを 1 つ追加する。
