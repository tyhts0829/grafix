# Pinterest Pen Plotter Character Art Loop Plan

## 目的

Pinterest の pen plotter ボードから、キャラクターの異なる参照アートを選び、抽象化して新しいアートとして再解釈した画像を 10 枚生成する。生成画像を `sketch/work/4.PNG` から連番で保存し、それぞれを Grafix で再現する `sketch/work/*.py` を作成する。

## 前提

- 既存の作業ツリー差分には触れない。
- Pinterest / ChatGPT の UI 操作は Safari と Computer Use を使って行う。
- Pinterest のスクリーンショットは作品単位で個別に切り抜き、画像生成時は毎回異なる 2〜3 枚を個別ファイルとして添付する。
- ChatGPT image generation の入力文は「抽象化してから新たなアートとして昇華して。アクセントカラーは1色のみ使用可能」のみとする。
- 生成画像は pen plotter で描画できる線画として昇華し、スクリプト側では塗りつぶしで隠す表現を避ける。
- トポロジー的な穴・切り取りが必要な箇所は `E.clip` を使う。
- 文字要素と塗りつぶしは使用してよい。
- 依存追加や破壊的操作は行わない。

## アクション

- [x] Safari で Pinterest を開き、pen plotter ボードからキャラクターの異なる参照アートをスクリーンショット保存する。
- [x] Pinterest のスクリーンショットから、参照アートを個別画像として切り抜く。
- [ ] 参照から任意の 2〜3 枚を個別に添付し、「抽象化してから新たなアートとして昇華して。アクセントカラーは1色のみ使用可能」という入力文だけで ChatGPT image generation を実行する。
- [ ] 毎回 Pinterest の参照アートから異なる 2〜3 枚を選び直して再生成し、最終的に 10 枚を確保する。
- [ ] 10 枚の生成画像を `sketch/work/4.PNG` から連番で保存する。
- [x] 既存の `sketch/work/*.py` と Grafix API の書き方を確認する。
- [ ] 各生成画像の特徴を抽象化し、保存した画像番号に対応する Grafix スクリプトを作成する。
- [ ] 穴・切り取り・重なりの除外が必要な表現では `E.clip` を使い、線で塗りつぶして隠す実装を避ける。
- [ ] 文字表現や塗りつぶしを、生成画像の再現に必要な範囲で使う。
- [ ] `PYTHONPATH=src python3 -m grafix export ...` で各スクリプトをレンダリング確認する。
- [ ] 必要に応じてスクリプトを調整し、完了・未完了項目をこの計画ファイルに反映する。
