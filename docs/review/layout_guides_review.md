現状のプリセットは「正方形グリッド」「比率ベースの再帰分割（左右／上下から）」「貴金属比矩形のタイル分割」という、構図の当たりを付けるには十分に強い 3 本柱になっています。一方で、Pinterest でよく見る“レイアウト技術”寄り（特にスイス系のタイポ／エディトリアル）を本気で支えるには、いくつかの「欠けている定番ガイド」があります。ここでは、追加コストに対して効果が大きい順に、機能として加える価値が高いものを挙げます。

まず最優先で入れた方がいいのは、マージン（安全域）とトリム／ブリードのガイドです。現状の border は「キャンバス外枠」しか表現できず、実際のレイアウト検討で一番頻出する“文字や主要要素を入れてはいけない領域”が可視化できません。レイアウトの良し悪しは、配置以前に「余白の規律」で決まることが多いので、margin_l, margin_r, margin_t, margin_b（最低でも左右上下別）で内側の矩形を引けるようにすると、ガイドの実用性が一段上がります。ブリードやトリムも同様に「外側／内側に何 mm 相当」を線で出すだけなので、実装は \_canvas_border の“内接版”を 1 本足すだけで済みます。さらに言うと、ここまで入れるなら「内側矩形だけを対象に後続のガイドを描く」オプション（例：use_safe_area: bool）も用意すると、ガイド線が“実際に使う領域”に集中して判断が速くなります。

次に効果が大きいのは、正方形グリッドではなく「カラム＋ガター（列組）」と「モジュラーグリッド（行 × 列）」です。あなたの square は汎用ですが、エディトリアル／ポスター系の実務的な構成検討では「12 カラム」「6 カラム」「縦方向はベースラインに合わせる」という発想が中心で、正方形セルに頼るとガター設計や列幅の規律が作りづらい。追加するなら、cols, rows, gutter_x, gutter_y で計算して、マージン内に収まる列境界線（および必要なら列中心線）を引くパターンを一つ入れるのが筋が良いです。これがあるだけで、タイポ主体のレイアウト検討が急に“それっぽく”ではなく“再現可能”になります。

その次に入れる価値が高いのは、ベースライングリッド（行送り）です。あなたのガイドは現在「形の割り付け」には強いのに「文字の縦リズム」を作る道具がありません。特にスイス寄りの整い方を狙うなら、縦の基準（ベースライン）無しで詰めていくのは遠回りになります。実装は単純で、baseline_step と baseline_offset（または safe area の上端からのオフセット）で水平線を繰り返すだけです。ここで重要なのは、axes="horizontal" と組み合わせたときに「ベースラインだけを表示できる」ことと、マージン内だけに線を出せることです。ガイド線が増えすぎると逆に判断が遅くなるので、表示範囲を制限できる設計が効きます。

四番目は、頻出の構図ガイドを“即出せる”ようにすることです。具体的にはセンター（縦横の中心線）と、三分割（1/3・2/3）と、黄金分割（0.382・0.618）のセットです。あなたの ratio_lines は黄金分割を表現できますが、現在は metallic_n 由来の比率に固定されているため、三分割をやろうとした瞬間に詰みます（metallic_mean では 1.5 が作れないので 1/3・2/3 が出せない）。ここは設計上の盲点で、ratio_lines は「metallic 専用」ではなく「任意比率」ツールにしておいた方が後で絶対に得します。UI 的には ratio: float を追加して、ratio_lines のときだけそれを使い、metallic_rectangles のときだけ metallic_n を使う、という割り切りで十分です。さらに、交点を“配置候補点”として見せたいなら、交点に小さな十字マークを打つオプション（例：show_intersections: bool, mark_size）があると、目視判断が速くなります。点のマーカーは線 2 本で表現できるので、プリミティブが G.line しかなくても実装できます。

五番目として、対角線とダイナミックシンメトリー系のガイドがあると、ミニマル構成の「斜めの緊張感」を作るときに効きます。少数要素の配置は、グリッドと比率だけだと“固すぎる／無難すぎる”に寄りやすく、対角線（四隅を結ぶ）と、必要なら「内接矩形の対角線（reciprocal）」を出せるだけで、配置の選択肢が増えます。実装は単純に 2 本（または追加数本）の線なので、コストが低い割に効果が出ます。

六番目は、ガイドの「密度制御」です。ratio_lines は levels を上げると線数が爆発します。あなたの実装だと基本的に線位置数が概ね 3\*\*levels - 1 のオーダーで増えるので、levels=8 で片軸だけでも 6560 本近くになり得ます。これが Geometry の + 連結やレンダリングに効いてくると、UI の操作感が一気に落ちます。ここは機能として min_spacing（隣接線がこれ未満なら打たない）か max_lines（上限を超えたら打ち切る）か、あるいは \_ratio_positions 側で「セグメント長が閾値未満なら分割しない」を入れておくのが現実的です。見た目としても、一定以下の間隔のガイドは情報量が過剰で判断を鈍らせるので、密度制御は単なる最適化ではなく“レイアウト支援の品質”そのものです。

七番目は、ガイド線の「階層化（メジャー／マイナー）」です。表示系の都合が許すなら、例えば「大区画の線」「小区画の線」を分けて、z をずらす、もしくは別 Geometry として返す（ビューア側で色や太さを変えられる前提）と、ガイドが“読める”状態になります。現状だと全線が同格なので、線が増えるほど視認性が下がります。ペンプロッタ用途で実際に描く可能性があるなら、さらに重要で、同じ線が重複して引かれないようにする（例えば border=True のときに square の端線を除外する）といった重複排除も実質的な機能になります。

最後に設計面の指摘ですが、いまの pattern が単一選択になっているため、「正方形グリッド＋黄金分割＋マージン」のような“重ね合わせ”を 1 回の呼び出しでやろうとすると設計が苦しくなります。UI の仕組み上マルチセレクトが難しいなら、落とし所として「頻出のオーバーレイは bool で別枠にする」のが現実的です。例えば show_center, show_thirds, show_margins, show_diagonals を pattern と独立に持たせ、pattern は“主ガイド”だけを決める、という分離です。これなら UI は壊さずに、実際の作業で必要な組み合わせがほぼカバーできます。

まとめると、このプリセットに足すなら「マージン／トリム／ブリード」「カラム＋ガター／モジュラー」「ベースライン」「センター／三分割／黄金分割の即時オーバーレイ」「対角線」「密度制御」「メジャー／マイナー階層」が、追加コストに対して効果が大きいです。特に ratio_lines を貴金属比に固定している点は、用途（レイアウト技術の学習・検討）とズレるので、任意 ratio に切り替えられるようにするのが最初の一手として妥当です。
